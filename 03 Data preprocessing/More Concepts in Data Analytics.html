We will first review some important concepts in data analytics. Below is an example of a dataset. Let's assume our task is to predict the employee's productivity based on other given information.

- Dataset: the collection of data. In this example, the whole data table.

- Data instance: one object in the dataset. Also referred to as <b>row</b> (mostly in 2D data), <b>data point</b>, <b>instance</b>, <b>record</b> (in database). In this example, each employee is one instance.

- Attribute: one characteristic of the data. Also referred to as <b>column</b> (mostly in 2D data), <b>variable</b>. In this example, Employee ID, Name, Salary, Experience... are attributes, or columns.

- Target: The column that is the goal of the analysis, usually what we want to predict. In this case, Productivity is the target. Also called target variable, dependent variable. Recall, we only> have a target in supervised learning tasks.

- Input, or , input columns, or features: columns that we give a model to perform the given task that are not the target. Note that not all columns in a dataset are useful for a model. For example, in this case, ID and Name are not useful to predict employee's productivity. However, in analysis where differences among people are emphasize, ID can be a good input.
    
    - Numeric column: columns of which values are meaningful numbers. This means that mathematical operations like adding, subtracting, averaging, etc., yield meaningful results. For examples, age is a numeric column, because average age is meaningful. Team number, e.g., team 1, team 2... is not a numeric column even though it appears as numeric, because average team number does not mean anything.
    
    - Class column: columns of which values are not numeric. They usually are string values, but in certain cases still look like numbers. The difference from a true numeric column is that math operations/functions applied on numeric columns are meaningful. For example, Salary is a numeric column, mean/average of salary makes sense. If we convert Department to number (e.g. HR = 1, IT = 2, R&D = 3), it is still a class column, because the average of department codes is not meaningful.
    
        - Ordinal class: classes of which values have inherent order. These columns can be convert to meaningful numbers. For example, Experience values of Low - Medium - High can be converted to 1 - 2 - 3, because Low < Medium makes sense, and a mean experience of 1.5 suggests more employees are in low and medium experiences.
        
        - Nominal class: classes of which values have no inherent order. These columns cannot be convert to meaningful numbers. For example, Department and Branch. HR > IT is meaningless.
        
        - Binary class: classes which have exactly two columns. They are the easiest to deal with. They can be converted to 0 an 1 without worrying about meaning of the numeric values.
        
- Distribution: how values of a column are distributed. In a numeric columns, we may look at the mean, standard deviation, shape of histograms... In a class column, we usually look at the frequency of values.

- Missing value: fields in data that have no meaningful values compared to other fields in the same column. Note that depend on the data provider, missing values may or may not be an empty value in the data. For example, they can be a "?" instead of an empty string. Certain companies code their missing value with a number, -1, or 99. We need to refer to the data documentation to make sure about the used format.
